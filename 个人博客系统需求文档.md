# 个人博客系统需求文档
## 一、项目概述
### 1.1 项目背景
在信息快速传播的时代，个人博客已经成为人们分享知识、记录生活和展示自我的重要平台。为了满足用户对高质量博客系统的需求，我们决定开发一款功能完善、界面美观、性能优良的个人博客系统。
### 1.2 项目目标
本个人博客系统旨在为用户提供一个简洁、易用、高效的博客平台，使用户能够方便地发布、管理博客文章，与读者进行互动交流。同时，系统管理员能够对博客进行全面的管理和维护，确保系统的稳定运行。
## 二、功能需求
### 2.1 用户管理
- 用户注册
  - 支持邮箱注册，提供验证码验证功能。
  - 密码需进行加密存储，确保用户信息安全。
  - 注册成功后自动跳转到登录页面。
- 用户登录
  - 支持邮箱和密码登录。
  - 提供记住登录状态功能，方便用户下次访问。
  - 登录失败时给出明确的提示信息。
- 个人信息管理
  - 用户可以修改个人基本信息，如昵称、头像、个人简介等。
  - 支持修改登录密码，修改时需验证原密码。
### 2.2 文章管理
- 文章发布
  - 提供富文本编辑器，支持图片上传、代码高亮、格式排版等功能。
  - 文章支持分类和标签，方便用户对文章进行组织和检索。
  - 支持文章预览功能，确保文章发布前的效果符合预期。
- 文章编辑和删除
  - 用户可以对已发布的文章进行编辑和修改。
  - 支持文章的删除操作，删除前需进行确认提示。
- 文章列表和详情
  - 展示文章列表，支持按分类、标签、时间等条件进行筛选。
  - 点击文章标题可查看文章详情，显示文章内容、作者信息、发布时间等。
### 2.3 评论管理
- 评论功能
  - 读者可以对文章进行评论，评论内容需经过审核才能显示。
  - 支持评论的回复功能，形成评论互动。
- 评论管理
  - 文章作者可以对评论进行管理，包括审核、删除等操作。
  - 系统管理员可以对所有评论进行管理。
### 2.4 分类和标签管理
- 分类管理
  - 支持创建、编辑和删除文章分类。
  - 分类支持层级结构，方便用户对文章进行更细致的组织。
- 标签管理
  - 支持创建和管理文章标签。
  - 标签云展示，直观显示热门标签。
### 2.5 系统管理
- 用户管理
  - 系统管理员可以管理所有用户信息，包括查看、禁用、删除用户等。
  - 系统设置
  - 支持博客基本信息的设置，如博客名称、副标题、SEO 设置等。
  - 支持网站主题的切换和自定义设置。
- 数据统计
  - 提供文章阅读量、评论数等数据的统计和分析功能。
  - 展示网站访问量、用户活跃度等数据图表。
## 三、非功能需求
### 3.1 性能需求
  - 系统响应时间在合理范围内，页面加载时间不超过 3 秒。
  - 支持高并发访问，在峰值情况下系统仍能稳定运行。
### 3.2 安全需求
  - 用户信息加密存储，防止数据泄露。
  - 对用户输入进行过滤和验证，防止 SQL 注入、XSS 攻击等安全漏洞。
  - 系统支持数据备份和恢复功能，确保数据的安全性和可靠性。
### 3.3 可用性需求
  - 系统界面简洁明了，操作方便快捷。
  - 提供完善的帮助文档和使用指南，方便用户使用。
### 3.4 可扩展性需求
  - 系统架构设计具有良好的可扩展性，方便后续功能的添加和升级。
  - 支持插件机制，允许用户根据自己的需求安装和使用插件。 
## 四、数据需求
### 4.1 数据库设计
###### 用户表（user）
  id：用户 ID，主键<br>
  username：用户名<br>
  email：邮箱<br>
  password：密码（加密存储）<br>
  avatar：头像<br>
  introduction：个人简介<br>
  create_time：创建时间<br>
  update_time：更新时间<br>
###### 文章表（article）
  id：文章 ID，主键<br>
  title：文章标题<br>
  content：文章内容<br>
  summary：文章摘要<br>
  category_id：分类 ID，外键<br>
  user_id：作者 ID，外键<br>
  status：文章状态（已发布、草稿、已删除）<br>
  view_count：阅读量<br>
  comment_count：评论数<br>
  create_time：创建时间<br>
  update_time：更新时间<br>
###### 分类表（category）
  id：分类 ID，主键<br>
  name：分类名称<br>
  parent_id：父分类 ID<br>
  create_time：创建时间<br>
  update_time：更新时间<br>
###### 标签表（tag）
  id：标签 ID，主键<br>
  name：标签名称<br>
  create_time：创建时间<br>
  update_time：更新时间<br>
###### 文章标签关联表（article_tag）
  id：关联 ID，主键<br>
  article_id：文章 ID，外键<br>
  tag_id：标签 ID，外键<br>
###### 评论表（comment）
  id：评论 ID，主键<br>
  content：评论内容<br>
  article_id：文章 ID，外键<br>
  user_id：用户 ID，外键<br>
  parent_id：父评论 ID<br>
  status：评论状态（已审核、未审核、已删除）<br>
  create_time：创建时间 <br>
### 4.2 Redis 缓存设计
  - 缓存热门文章列表，提高访问速度。
  - 缓存文章详情，减少数据库查询。
  - 缓存用户会话信息，提高用户体验。
## 五、技术实现方案
###  5.1 技术栈选择
  - 后端：Spring Boot、MyBatis-Plus、MySQL、Redis
  - 前端：Vue.js、Element UI
  - 部署：Docker、Nginx
###  5.2 系统架构
  - 表现层：负责与用户交互，接收用户请求并返回处理结果。
  - 业务逻辑层：负责处理业务逻辑，调用数据访问层获取数据。
  - 数据访问层：负责与数据库交互，实现数据的增删改查操作。
  - 缓存层：使用 Redis 缓存常用数据，提高系统性能。
## 六、项目进度计划
###  6.1 需求分析和设计阶段（第 1-2 周）
  - 完成需求分析和整理

  - 搭建开发环境

  - 实现文章管理功能
  - 实现评论管理功能
  - 实现分类和标签管理功能
  - 实现系统管理功能
###  6.3 测试阶段（第 9-10 周）
  - 进行单元测试
  - 进行集成测试
  - 进行性能测试
  - 修复测试中发现的问题

  - 进行系统部署
  - 进行数据迁移
  - 进行上线前的最终测试
  - 正式上线运行
